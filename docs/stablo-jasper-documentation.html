1. Project Overview & Architecture

Stablo Jasper is a headless blog platform where WordPress manages content and a Next.js 14 application serves the public-facing website. The two systems communicate exclusively through WPGraphQL — a GraphQL API layer installed on WordPress.

4. Data Flow — How It All Works

Page Load (Static Generation)

1. Build time: Next.js calls generateStaticParams()
   → queries WPGraphQL for all post slugs
   → generates one static HTML file per post

2. User visits /post/my-article
   → CDN serves pre-built HTML instantly (no WordPress involved)
   → Page loads in <100ms

3. Cache expires (after 3600s / 1 hour)
   → Next.js background-regenerates the page from fresh WP data
   → Users never see a loading state

On-Demand Revalidation (Webhook)

1. You publish/update a post in WordPress
   → WP fires: POST https://yoursite.com/api/revalidate
      with header: Authorization: Bearer REVALIDATE_SECRET
      with body: {"type":"post","slug":"my-article"}

2. Next.js validates the secret
   → Calls revalidatePath('/post/my-article') + revalidateTag('post-my-article')

3. Next request to that URL fetches fresh data from WPGraphQL
   → New static HTML built and cached
   → Visitors see updated content within seconds

Image Delivery

WordPress media URLs pass through Next.js's built-in image optimiser (next/image), which automatically converts to WebP / AVIF, applies lazy loading, and sizes images correctly for the viewport. No CDN plugin needed on WordPress for images.

7. ISR Webhook — On-Demand Revalidation

What is ISR?

Incremental Static Regeneration (ISR) means pages are pre-built as static HTML (fast, no server cost) but can be refreshed on-demand without a full redeploy. When you publish a post, WordPress tells Next.js which page to rebuild — that specific page is regenerated in the background within seconds.

WordPress Setup (functions.php)

Add this to your active theme's functions.php or a custom plugin file:

add_action('save_post', function($post_id, $post) {
    if ($post->post_status !== 'publish') return;
    if (wp_is_post_revision($post_id)) return;

    wp_remote_post('https://yoursite.com/api/revalidate', [
        'headers' => [
            'Content-Type'  => 'application/json',
            'Authorization' => 'Bearer ' . YOUR_REVALIDATE_SECRET,
        ],
        'body'     => json_encode([
            'type' => 'post',
            'slug' => $post->post_name,
        ]),
        'blocking' => false,   // fire-and-forget, doesn't slow WP save
    ]);
}, 10, 2);

8. Security Architecture

HTTP Security Headers (next.config.js)

HeaderValue / Purpose
Content-Security-PolicyRestricts scripts, images, frames to known origins. Whitelists Google AdSense domains.
X-Frame-OptionsSAMEORIGIN — prevents clickjacking
X-Content-Type-Optionsnosniff — prevents MIME-type sniffing
Referrer-Policystrict-origin-when-cross-origin
Permissions-PolicyDisables camera, microphone, geolocation
Strict-Transport-SecurityForces HTTPS for 2 years (preload)

Authentication (JWT)

  • JWT tokens are issued by WordPress via WPGraphQL JWT Auth plugin
  • Tokens are stored in httpOnly cookies — never in localStorage (XSS-safe)
  • Cookie is SameSite=Strict — CSRF-safe
  • Token validation happens server-side on /api/auth/me
  • Never forward WordPress error messages to the client

Input Validation

  • Every API route body is validated with Zod before processing
  • Slug parameters sanitised (strip non-alphanumeric/hyphen) before GraphQL calls
  • Search queries limited to 100 chars and sanitised with regex
  • Comment content length capped at 5000 characters

Spam Protection

  • All comment submissions run through Akismet API before being sent to WordPress
  • Spam comments silently return success (don't tip off spammers)
  • Login rate limit: 5 attempts per 15 minutes per IP
WPGraphQL Introspection Disable public introspection in WP Admin → GraphQL → Settings in production. This hides your schema from potential attackers.

9. SEO Setup & Schema Markup

Per-Page Metadata (generateMetadata)

Every page exports a generateMetadata() function that sets:

  • Title, description, canonical URL
  • OpenGraph tags (title, description, image, type: article)
  • Twitter Card (summary_large_image)
  • Published/modified dates for articles
  • If Yoast plugin installed: pulls all fields directly from Yoast via GraphQL

JSON-LD Schema Markup (components/seo/JsonLd.tsx)

PageSchema Types Injected
HomepageWebSite (with SearchAction), Organization
/post/[slug]BlogPosting, BreadcrumbList, Person (author)
/category/[slug]CollectionPage, BreadcrumbList
/author/[slug]ProfilePage, Person
All pagesBreadcrumbList

BlogPosting Schema Fields

  • headline, description, url, datePublished, dateModified
  • author (name + url), publisher (name + logo), image (url + dimensions)
  • keywords (from categories), articleSection (first category)
  • mainEntityOfPage (WebPage)

Sitemap

Generated automatically after every build via next-sitemap. Includes all published posts, category pages, and author pages with lastmod from WordPress's post.modified date.

Core Web Vitals

  • Hero/featured images use priority prop — preloaded, no LCP penalty
  • All other images are lazy-loaded with automatic WebP/AVIF conversion
  • Ad slots have CSS min-height reserved — no Cumulative Layout Shift (CLS)
  • Heavy components (Comments, Ads, Newsletter) loaded via next/dynamic

11. Comments System

File: components/blog/Comments.tsx

How It Works

User submits comment form
→ POST /api/comments (validated with Zod)
→ Akismet spam check (server-side)
→ If clean: POST to WordPress REST API /wp-json/wp/v2/comments
→ Comment appears in WP Admin as pending moderation
→ After admin approves: shows in comments section

Features

  • Threaded/nested comments (WordPress native)
  • Reply-to with parentId support
  • Author name, email (required), website URL (optional)
  • Optimistic UI — pending message shown immediately
  • Gravatar avatars from WordPress
  • Form validation with react-hook-form

12. Newsletter (Mailchimp)

File: components/blog/NewsletterForm.tsx

The newsletter form submits to /api/newsletter, which proxies to Mailchimp's API server-side (your API key is never exposed to the browser). Uses double opt-in — subscribers receive a confirmation email before being added (GDPR compliant).

Placement

  • After post body (full form with first name field)
  • Sidebar (compact single-line form)
  • Footer (compact)

13. Subscriber-Gated Content

Files: components/auth/GatedContent.tsx, components/auth/LoginModal.tsx

How It Works

  1. In WordPress: add ACF boolean field is_subscriber_only to the post
  2. Set it to true for premium posts
  3. Next.js reads the field via GraphQL (requires ACF + WPGraphQL for ACF plugins)
  4. If true: shows first 2 paragraphs as teaser + blur gradient + login CTA
  5. User clicks "Sign In" → Login Modal appears → submits to /api/auth/login
  6. On success: JWT stored in httpOnly cookie → full post content revealed
Enable Feature Flag Set NEXT_PUBLIC_ACF_WPGRAPHQL=true in .env.local once ACF and WPGraphQL for ACF plugins are installed.

14. Push Notifications

Files: components/push/OneSignalInit.tsx, components/push/WebPushrInit.tsx

How It Works

  • Both SDKs are loaded once via next/script strategy="lazyOnload" in the root layout
  • They show the browser's native push permission prompt after 30 seconds
  • When you publish a post in WordPress, the WP plugin (OneSignal / WebPushr) automatically sends a push notification to all subscribers
  • No custom API route needed in Next.js — all handled by the WP plugins

Deployment (Vercel — Recommended)

  1. Go to vercel.com → New Project → Import from GitHub → select stablo-jasper
  2. Add all environment variables from .env.local into Vercel's Environment Variables section
  3. Deploy — Vercel auto-detects Next.js
  4. Update WP_GRAPHQL_URL, WP_SITE_URL, and SITE_URL to your production WordPress domain
  5. Update the revalidate webhook URL in WordPress functions.php to your Vercel domain
ISR on Vercel On-demand revalidation (/api/revalidate) works natively on Vercel without any extra configuration. On other hosts, ensure your Next.js server stays running (not just static export).
